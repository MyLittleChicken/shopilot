# 아키텍처

## 표면과 신뢰 경계

화면(web·widget)은 프레젠테이션만 맡는다. LLM 키와 에이전트 파이프라인, 데이터 접근은 전부 서버에 둔다. 위젯은 외부 사이트에 심기는 신뢰할 수 없는 클라이언트이므로, 시크릿이 클라이언트로 새어 나가서는 안 된다(상세: `.claude/security.md`).

양면 구조를 지향한다.

- 구매자(B2C)는 자체 웹(`apps/web`)이나 판매자 쇼핑몰에 설치된 위젯(`apps/widget`)에서 Shopilot과 대화한다.
- 판매자(B2B)는 자기 쇼핑몰에 위젯을 `<script>` 한 줄로 설치한다.

## 시스템 구조

```
packages/ui — 공유 UI 컴포넌트 (컨테이너 반응형)
   │  두 표면이 같은 컴포넌트를 쓴다
   ├─▶ apps/web      풀페이지 웹 (디자인 원본 · 데모)
   └─▶ apps/widget   외부 사이트에 <script>로 임베드 (Shadow DOM)
              │
              │  데이터 소스를 어댑터로 교체
              ├─ (개발) 목 데이터 — 서버 없이 UX 선구현
              └─ (운영) apps/server — SSE 스트리밍
                          ├─ 에이전트 파이프라인 (단일 · 카테고리 프로파일/프롬프트로 분기, 가전 먼저)
                          ├─ LLM 어댑터 (Claude / OpenAI / Gemini 교체)
                          └─ 데이터 어댑터 (시드 카탈로그 ↔ 실제 커머스 API)
```

| 워크스페이스 | 역할 |
| --- | --- |
| `apps/server` | 백엔드 API. 에이전트 파이프라인, LLM·데이터 어댑터, 시크릿, SSE 스트리밍 |
| `apps/web` | 자체 풀페이지 웹. 디자인을 만들고 확인하는 표면이자 데모 |
| `apps/widget` | 외부 쇼핑몰에 임베드되는 위젯. `packages/ui`를 Shadow DOM으로 감싸 노출 |
| `packages/ui` | 공유 UI 컴포넌트. web·widget이 같은 디자인을 쓴다 |
| `packages/core` | 에이전트 파이프라인과 어댑터, 실행 진입점(`runAgent`) |
| `packages/schemas` | 공유 Zod 스키마 (상품·메시지·요청·이벤트) |

## 설계 원칙

1. 외부 의존성은 어댑터 뒤에 둔다. LLM(Claude·OpenAI·Gemini)과 데이터 소스를 같은 인터페이스로 추상화해, 코어 로직이 구현체를 모르게 한다.
2. 데이터는 목 → 시드 → 실데이터 순으로 전환한다. 서버 없이 목 데이터로 UI를 먼저 완성하고, 같은 인터페이스로 실데이터에 연결한다. 덕분에 데모는 항상 동작한다.
3. 에이전트 파이프라인은 하나로 유지한다. 카테고리 차이는 별도 코드가 아니라 **카테고리 프로파일** — 시스템 프롬프트, 비교 기준(중요 속성), 도구·데이터 설정 — 로 다룬다. 새 카테고리는 프로파일과 데이터만 추가해 확장하고, 흐름이 크게 달라지는 경우는 별도 파이프라인 대신 LLM의 도구 호출(agentic)로 분기시킨다. 가전/전자제품을 첫 대상으로 잡는다.
4. 스키마를 먼저 정의한다. 상품·메시지·요청·이벤트를 Zod로 정의하고 런타임에 검증한다. 클라이언트와 서버가 같은 스키마를 공유한다.
5. UI는 `packages/ui` 한 곳에만 만든다. 컨테이너 쿼리로 작성해 넓은 웹 화면과 좁은 위젯 패널 양쪽에 코드 한 벌로 대응한다.
6. 시크릿은 서버에만 둔다.

## LLM 응답 품질

LLM 응답은 실서비스에서 흔들리기 쉬우므로 검증 계층을 둔다. 응답 포맷을 Zod로 검증하고, 실패하면 폴백·재시도하며, 이상 입력에 대응한다.

## 기술 스택

- 모노레포: pnpm workspace + Turborepo
- 서버: Hono + SSE (Node 런타임)
- 웹: Next.js
- 위젯: Vite로 단일 번들 빌드, Shadow DOM 격리, CDN 배포
- 공유 UI: `packages/ui` — 스코프드 CSS와 컨테이너 쿼리로 임베드 환경에서 견고하게 둔다. 앱 단위로 Tailwind는 선택적으로 쓴다.
- 계약: Zod (`packages/schemas`)
- 데이터/인프라: Supabase(Postgres) 검토 중
- 언어: 한국어 우선, i18n 여지를 남긴다

LLM 프로바이더는 어댑터 뒤에서 교체하며, 기본 선택은 실제 연동 시점에 정한다.
