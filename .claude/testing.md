# 테스트

아직 테스트 코드는 없다. 도입할 때 아래 원칙을 따른다.

- 타입체크가 기본 게이트다 — `pnpm -r typecheck`는 항상 통과해야 한다. CI는 타입체크 → 린트 → 테스트 순으로 돌린다.
- 핵심 로직과 경계 조건을 테스트한다. 자명한 코드나 구현 세부는 테스트하지 않는다.
- 외부 의존성(LLM·네트워크·시간·랜덤)은 어댑터 뒤에 격리하고 목으로 대체해 테스트를 결정론적으로 유지한다.
- 데이터 모양과 계약은 `packages/schemas`로 검증하고, 예시 픽스처로 회귀를 막는다.
- 비결정론적 출력은 결과의 내용이 아니라 구조·동작을 검증한다.
- 러너는 vitest를 우선 검토한다. 테스트는 대상 가까이 `*.test.ts`로 둔다.
