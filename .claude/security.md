# 보안

## 원칙

- 시크릿(LLM 키, DB 자격증명 등)은 서버(`apps/server`)에서만 `process.env`로 읽는다.
- 클라이언트(web·widget·ui)에는 공개 변수만 둔다 — `NEXT_PUBLIC_*`, `VITE_*`.
- 위젯은 외부 사이트에 임베드되는 신뢰할 수 없는 클라이언트다. 비즈니스 로직과 시크릿을 클라이언트 번들에 넣지 않는다.
- 키를 코드에 하드코딩하지 않는다. 실제 값은 `.env`(커밋 금지)에 두고, `.env.example`에는 플레이스홀더만 남긴다.

## 자동 가드 (`.claude/hooks`)

파일 수정·git 작업 시 자동으로 검사한다. 보조 장치이며, 1차 방어는 `.gitignore`와 서버/클라이언트 코드 분리다.

- 클라이언트 코드에 서버 시크릿이 들어가면 차단.
- 하드코딩된 키(Anthropic·OpenAI·Google·AWS·GitHub·JWT 등)를 차단.
- `.env` 스테이징과 광역 `git add`(`.`/`-A`)를 차단.
- 커밋 직전 스테이징된 내용에서 비밀값을 스캔.

## 커밋 금지

- `.env*` — 단, `.env.example` 템플릿은 허용한다. 실제 값은 커밋하지 않는다.
